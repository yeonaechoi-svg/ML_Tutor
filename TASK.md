## MVP (2일 완성 목표 — 수업 즉시 투입 가능)

### MVP-2. 데이터베이스 모델 설계

**작업 내용**
app/models.py에 아래 모델 작성

1. User 모델
```python
- id (PK)
- name (학생 이름)
- student_id (학번, unique)
- class_name (반, 예: "1반")
- password_hash
- role ('student' / 'teacher')
- is_first_login (Boolean, default=True)
- created_at
```

2. Progress 모델
```python
- id (PK)
- user_id (FK → User)
- stage (1~7, 현재 단계)
- substep (단계 내 세부 스텝 번호)
- status ('in_progress' / 'completed')
- updated_at
```

3. ChatLog 모델
```python
- id (PK)
- user_id (FK → User)
- stage (1~7)
- substep
- question (AI가 던진 질문)
- answer (학생 답변)
- feedback (AI 피드백)
- created_at
```

4. QuizResult 모델
```python
- id (PK)
- user_id (FK → User)
- stage
- score (맞은 개수)
- total (전체 문항 수)
- created_at
```

5. PeerReview 모델 (MVP에서는 모델만 생성, 기능은 v0.3에서 구현)
```python
- id (PK)
- reviewer_id (FK → User)
- reviewee_id (FK → User)
- checklist_json (JSON 문자열)
- created_at
```

**완료 기준**
- `flask db init` 없이 앱 실행 시 DB 자동 생성
- instance/ml_tutor.db 파일 생성 확인

### MVP-3. 로그인 / 인증 기능

**작업 내용**
app/auth/routes.py 작성

1. 학생 로그인 페이지 (GET /login)
- 반 선택 드롭다운
- 학번 입력
- 이름 입력
- 비밀번호 입력
- 로그인 버튼

2. 로그인 처리 (POST /login)
- 반 + 학번 + 이름 + 비밀번호 검증
- 성공 시 Flask-Login session 저장
- is_first_login=True이면 비밀번호 변경 페이지로 리다이렉트
- 실패 시 오류 메시지 표시

3. 비밀번호 변경 페이지 (GET/POST /change-password)
- 새 비밀번호 입력
- 새 비밀번호 확인 입력
- 변경 완료 시 is_first_login=False 저장
- 학생 대시보드로 리다이렉트

4. 로그아웃 (GET /logout)

5. 교사 로그인
- 별도 교사 로그인 페이지
- .env의 TEACHER_CODE로 인증

**완료 기준**
- 학생 로그인 → 첫 로그인 시 비밀번호 변경 페이지 → 대시보드 이동
- 로그인 안 된 상태에서 학생 페이지 접근 시 로그인 페이지로 리다이렉트

### MVP-4. 교사 관리 페이지 — 학생 등록

**작업 내용**
app/teacher/routes.py 작성

1. 교사 대시보드 (GET /teacher/dashboard)
- 전체 학생 수
- 단계별 진행 학생 수 현황
- 학생 목록 테이블 (이름, 반, 학번, 현재 단계, 최종 접속일)

2. 학생 명단 엑셀 업로드 (GET/POST /teacher/upload)
- 엑셀 파일 업로드 폼
- 엑셀 형식: A열=반, B열=학번, C열=이름
- 업로드 시 User 모델에 자동 저장
- 초기 비밀번호: 1234 (해시 처리)
- is_first_login=True 설정
- 중복 학번 처리 (이미 있으면 건너뜀)

3. 학생 비밀번호 초기화 (POST /teacher/reset-password/<user_id>)
- 비밀번호를 1234로 초기화
- is_first_login=True로 변경

**엑셀 파일 형식 예시**
```
A열(반)   B열(학번)   C열(이름)
1반       20250101    홍길동
1반       20250102    김철수
2반       20250201    이영희
```

**완료 기준**
- 엑셀 업로드 후 학생 목록에 반영 확인
- 비밀번호 초기화 버튼 동작 확인

### MVP-5. 1단계 사전학습 콘텐츠

**작업 내용**
app/student/routes.py + templates/student/stage1/ 작성

1단계는 아래 5개 세부 스텝으로 구성

**스텝 1-1. 인공지능이란?**
- 인공지능 개념 설명 (교과서 기반)
- 지능 에이전트 개념
- 전통적 프로그래밍 vs 기계학습 차이
- 다음 버튼 → 스텝 1-2로 이동 + Progress 저장

**스텝 1-2. 기계학습이란?**
- 기계학습 개념 설명
- 기계학습이 필요한 이유
- 기계학습 동작 원리 (데이터 → 학습 → 모델 → 예측)
- 다음 버튼 → AI 체크포인트 1 발동

**[AI 체크포인트 1]**
- 질문 유형: 개념 확인 + 비교 이해
- 예시 질문:
  "전통적 프로그래밍과 기계학습의 차이를 자신의 말로 설명해보세요.
   그리고 기계학습이 더 적합한 상황은 어떤 경우인지 예를 들어보세요."
- 학생 답변 입력 → Claude API 호출 → 피드백 표시
- 피드백 확인 후 다음 스텝으로 이동

**스텝 1-3. 기계학습 유형**
- 지도학습 vs 비지도학습 개념
- 지도학습: 회귀(예측) / 분류
- 비지도학습: 군집(K-평균)
- 각 유형별 실생활 사례 카드 (클릭하여 상세 보기)
  - 회귀: 자동차 CO₂ 배출량 예측, 아이스크림 매출 예측
  - 분류: 계절 판별, 스팸 메일 분류, 가위바위보 인식
  - 군집: 고객 세분화, 편의점 위치 군집화
- 다음 버튼 → AI 체크포인트 2 발동

**[AI 체크포인트 2]**
- 질문 유형: 유형 판별
- 예시 질문:
  "회귀, 분류, 군집의 차이를 설명하고,
   각각 어떤 상황에서 사용하는지 실생활 예시를 들어 설명해보세요."

**스텝 1-4. 알고리즘별 사례 실습**
교과서 실습 3개를 단계별로 안내

실습 A — 선형 회귀 (자동차 CO₂ 배출량 예측)
- 문제 정의 → 데이터 탐색 → 모델 생성 → 성능 평가 순서로 코드 설명
- 각 코드 블록마다 "이 코드는 무엇을 하는가" 설명 포함
- 코드는 읽기 전용으로 표시 (복사 가능)

실습 B — 분류 (학생 성적 등급 분류)
- 로지스틱 회귀 + 의사결정나무 + KNN 알고리즘 소개
- 교과서 동일 코드 + 설명

실습 C — 군집 (K-평균)
- K-평균 알고리즘 원리 설명
- 교과서 동일 코드 + 설명

각 실습 후 → AI 체크포인트 발동

**[AI 체크포인트 3 — 실습 A 후]**
- 개념 질문: "train_test_split을 사용하는 이유가 무엇인지 설명해보세요."
- 코드 질문: "코드에서 test_size=0.3이 의미하는 것은 무엇인가요?"

**[AI 체크포인트 4 — 실습 B 후]**
- 개념 질문: "분류 모델에서 정확도(Accuracy)가 의미하는 것을 설명해보세요."
- 코드 질문: "X와 y를 나누는 이유는 무엇인가요?"

**[AI 체크포인트 5 — 실습 C 후]**
- 개념 질문: "군집 분석에서 K값을 결정하는 것이 왜 중요한가요?"
- 코드 질문: "KMeans(n_clusters=2)에서 n_clusters의 의미를 설명해보세요."

**스텝 1-5. 이해 확인 퀴즈**
- 5문항 4지선다
- 교과서 사례 기반 유형 판별 문제
- 즉각 피드백 (정오답 + 이유 설명)
- 점수 저장 → QuizResult 모델
- 완료 시 2단계로 이동 버튼 표시

**Progress 저장 규칙**
- 각 스텝 완료 시 Progress 업데이트
- 학생이 다시 접속하면 마지막 완료 스텝 다음부터 시작

**완료 기준**
- 1단계 전체 흐름 완주 가능
- AI 체크포인트 5개 모두 동작
- 퀴즈 점수 DB 저장 확인

### MVP-6. AI 튜터 API 연동

**작업 내용**
app/ai_tutor/routes.py + app/ai_tutor/prompts.py 작성

1. API 엔드포인트 (POST /ai-tutor/checkpoint)

- 응답: feedback 텍스트

```
당신은 고등학교 정보 교과 인공지능 단원을 가르치는 AI 튜터입니다.

[역할]
- 학생이 기계학습 개념을 스스로 이해하도록 돕습니다.
- 절대로 정답 코드를 직접 제공하지 않습니다.
- 힌트와 유도 질문으로 학생이 스스로 생각하게 합니다.

[어조]
- 차분하고 격식있게 말합니다.
- 고등학생 수준에 맞는 언어를 사용합니다.
- 틀린 답변에도 격려하며 방향을 제시합니다.

[피드백 구조]
1. 학생 답변에서 잘된 부분 언급
2. 보완이 필요한 부분 힌트 제공
3. 생각해볼 추가 질문 1개 제시
```

3. 단계별 체크포인트 질문 목록 (prompts.py에 딕셔너리로 관리)

**완료 기준**
- 학생 답변 입력 → Claude API 응답 → 화면에 피드백 표시
- ChatLog DB 저장 확인

### v0.2 — 2단계: 아이디어 구상 [핵심 단계]

**작업 내용**

2단계는 프로젝트 전체에서 가장 중요한 단계.
학생이 자신의 프로젝트 주제를 선정하고 AI 튜터와 대화하며 방향을 잡는 과정.

1. 주제 입력 폼
- "어떤 사회 문제를 데이터로 해결하고 싶은가요?" 자유 입력
- AI가 입력을 읽고 적합한 기계학습 유형 판별 퀴즈 생성

2. AI 유형 판별 유도
- 학생이 입력한 주제를 바탕으로
- "이 문제는 수치를 예측하나요, 아니면 어떤 그룹에 속하는지 판별하나요?"
- 학생이 스스로 회귀/분류/군집을 결정하도록 유도

3. 알고리즘 추천
- 학생이 유형을 결정하면 AI가 적합한 알고리즘 추천
- 추천 이유 설명 (정답 제시 아님, 이유를 함께 생각하게 유도)

4. 아이디어 요약 저장
- 주제, 선택 유형, 알고리즘, 예상 데이터 소스 저장
- 3단계에서 이 내용을 바탕으로 진행

**완료 기준**
- 학생이 주제 입력 → AI와 대화로 유형/알고리즘 결정 → 요약 저장

**작업 내용**

- 통합 체크리스트형 자기평가 문항 (AI 생성)
